# Jira Issue

>  Issue별로 정리를 하고, Epic/Story/Task로 나누었습니다.

>  Epic은 기능 단위, Story는 각 기능의 CRUD 및 페이지 단위로 작성, 마지막으로 Task는 페이지 안에 세부적인 내용들 (+오류)를 기준으로 작성하였습니다.
